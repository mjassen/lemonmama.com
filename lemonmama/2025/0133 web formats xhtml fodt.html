Web Page Formats Testing Xhtml Pdf Fodt Etcetera

 

 

 

 

 

 

Spoiler alert. Want to publish a convenient rich web format and looks like I’m choosing .xhtml as the final format.

 

Details are as follows.

 

Wanting to push to switch over my blog posts to be within single files but still include images.

Also some more requirements.

 

Main requirements:

 

* Blog post contained in a single file.

* Blog post fully plaintext searchable, like via kedit or grepwin search.

* This plaintext search includes utf-8 characters, including Chinese language characters.

* Can include images within the blog post, like embed jpegs.

* Blog post comes up directly in all popular web browsers, means doesn't prompt download.

* Blog post file can be re-saved and re-edited ongoing as needed.

And so in the end of the test.

 

I will probably from here on out be publishing my blog posts in ".xhtml" format.

As it fits 5/6 of these.

The one it doesn’t fit is it is clunky to re-open re-edit it.

Sadly i have a lot of requriements.

.fodt fit 5/6 too. .Fodt doesn’t directly come up in browser.

.rtf fit 5/6 too. .Rtf isn’t utf-8 compatible sadly. So no utf-8 Chinese language character plaintext search for rtf.

.pdf fit 5/6 too. Not re-editable. Also I believe isn’t plain text searchable.

 

 

So either .xhtml or .fodt or .pdf it will be.

And if .xhtml then i guess I’ll hang on to a copy of the original document in .fodt format in case i need to re-open re-edit it later.

 

In conclusion, i did a lot of testing and found likely .xhtml format via LibreOffice Writer will be a main publishing tool for me going forward.
